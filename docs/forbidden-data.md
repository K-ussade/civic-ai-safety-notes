# Forbidden Data

## Purpose

This document lists data and operational material that must not be added to safety notes, examples, templates, reports, demos, or portfolio material.

## Never Publish

| Category | Examples | Required handling |
| --- | --- | --- |
| Donor data | Names, contact records, giving history, pledges, payment details. | Exclude. |
| Student data | Names, grades, attendance, learning records, guardian details, school identifiers tied to students. | Exclude. |
| Volunteer data | Contact details, schedules, background records, availability, private assignments. | Exclude. |
| School private data | Internal contacts, incident notes, contracts, non-public site details, private agreements. | Exclude. |
| Foundation private operations | Board notes, internal strategy, private finances, private incident records, unreleased plans. | Exclude. |
| Customer data | Customer records, tickets, contracts, telemetry, credentials, private business details. | Exclude. |
| Model material | Weights, checkpoints, sealed methods, private evaluations, hidden benchmarks, production prompts. | Exclude. |
| Dataset material | Raw datasets, private datasets, unapproved sanitized samples, source exports, private telemetry. | Exclude. |
| Infrastructure/security | Secrets, tokens, API keys, exact sensitive infrastructure locations, exploit details, security-sensitive NEURONA operational details. | Exclude. |
| Sealed IP | Sealed YOSO-YAi LLC IP, private training corpora, proprietary source, unreleased build methods. | Exclude. |

## Review Rule

If an artifact needs a private fact to sound credible, the public artifact is not ready. Replace the claim with status-labeled public-safe language or hold it for human review.

Required public-safe exclusion language includes donor data, student data, sealed YOSO-YAi LLC IP, and security-sensitive NEURONA operational details.
