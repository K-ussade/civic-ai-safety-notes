# Dataset Release Safety

## Purpose

This standard defines public-safe review requirements before a dataset card or dataset-release claim can be published.

## Required Review Areas

| Area | Required answer |
| --- | --- |
| Status | Is the dataset planned, scaffolded, released, paused, deprecated, or private/not-public? |
| Source | Is the source public, consented, licensed, or otherwise approved for release? |
| Privacy | Are donor data, student data, school private data, volunteer data, customer data, and private telemetry excluded? |
| Sanitation | Has any public sample passed documented sanitation review? |
| Governance | Has human data governance authority approved release? |
| Use limits | Are prohibited uses, redistribution limits, and context limits stated? |
| Linkage risk | Could records be re-identified through location, timing, combinations, or small groups? |
| Maintenance | Is correction, withdrawal, and deprecation handling documented? |

## Required Wording

- `Status: Planned / Not released` for future dataset ideas.
- `Status: Scaffolded / Not released` for dataset-card templates.
- `Status: Released` only when the public dataset artifact exists and human review approved publication.

## Non-Claims

Do not imply that a dataset has been collected, sanitized, uploaded, released, deployed, or approved unless each claim has public evidence and documented review.
