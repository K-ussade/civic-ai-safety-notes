# Claim Review Standard

## Purpose

This standard governs public technical claims in README files, model cards, dataset cards, Spaces, reports, profile repositories, and portfolio links.

## Review Table

| Claim type | Required evidence before publication |
| --- | --- |
| Planned | Public-safe roadmap or scaffold text. |
| Scaffolded | Public repo, template, or placeholder artifact that clearly says not released. |
| Released | Public artifact URL, review status, and authority approval. |
| Evaluated | Public evaluation summary or reviewed citation. |
| Deployed | Public deployment surface and human approval to name it. |
| Used by a school/program | Public approved source and privacy review. |
| Service offered | Alexandra review and monetization-readiness approval. |
| Impact/outcome | Reviewed evidence and public-safe attribution. |

## Required Checks

- Status is explicit.
- Evidence exists for every release or outcome claim.
- Entity owner and authority are named.
- Public/private/sealed boundaries are preserved.
- Foundation language is not used as company marketing.
- No fake models, datasets, Spaces, reports, schools, deployments, services, or outcomes are introduced.

## Safe Status Language

Use `planned`, `scaffolded`, `released`, `paused`, `deprecated`, or `private/not-public`. Do not use vague phrases such as `coming soon`, `live`, `production-ready`, or `impact-proven` without review.
