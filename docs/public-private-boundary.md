# Public And Private Boundary

## Purpose

This document defines what can and cannot be published in `civic-ai-safety-notes`.

## Allowed Public Material

| Material | Condition |
| --- | --- |
| Safety note templates | Clearly labeled as templates. |
| Limitation taxonomy | Public and non-operational. |
| Review gates | No private evaluation detail. |
| Public reliance boundaries | No private incident or sensitive abuse detail. |
| Artifact safety standards | General requirements only. |
| Public safety summaries | Approved after review. |
| Mermaid safety maps | No private workflow, security-sensitive details, or sealed process. |

## Forbidden Public Material

| Material | Reason |
| --- | --- |
| Sealed model methods | Company sealed IP boundary. |
| Private evaluations | Safety and release boundary. |
| Sensitive abuse details with operational value | Misuse prevention boundary. |
| Donor private data | Donor privacy boundary. |
| Student data | Student privacy boundary. |
| School private data | School and student privacy boundary. |
| Volunteer private data | Personal data boundary. |
| Private Foundation operations | Internal operating boundary. |
| Customer data | Company/customer confidentiality boundary. |
| Private training corpora | Privacy and release boundary. |
| Unreleased model weights | Release and safety boundary. |
| Private telemetry | Privacy and security boundary. |
| Production prompts | Security and sealed-method boundary. |
| Secrets, tokens, and API keys | Security boundary. |
| Security-sensitive NEURONA operational details | Operational safety boundary. |

## Generated Output Rule

Generated summaries, examples, screenshots, logs, model outputs, and diagrams inherit the boundary of their inputs until reviewed.
