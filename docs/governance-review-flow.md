# Governance Review Flow

## Purpose

This document defines the human review sequence before a public artifact or claim leaves planned/scaffolded status.

## Review Flow

| Step | Reviewer role | Required outcome |
| --- | --- | --- |
| 1. Artifact classification | Documentation owner | Model, dataset, Space, report, portfolio link, or repo copy is identified. |
| 2. Boundary review | Privacy/governance reviewer | Public, private, sealed, and mixed material is separated. |
| 3. Evidence review | Artifact owner | Public evidence exists for every release, evaluation, deployment, service, or impact claim. |
| 4. Safety review | Safety reviewer | Intended use, out-of-scope use, limitations, and reliance boundaries are acceptable. |
| 5. Publication review | Human Foundation authority | Status label and publication scope are approved. |
| 6. Post-publication monitoring | Maintainer | Corrections, withdrawals, and deprecations are tracked. |

## Stop Conditions

Stop publication when the artifact includes forbidden data, sealed IP, unsupported claims, private operations, security-sensitive NEURONA operational details, or ambiguous entity authority.
