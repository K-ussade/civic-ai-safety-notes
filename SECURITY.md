# Security Policy

## Reporting Sensitive Issues

Do not open a public issue containing secrets, private safety review records, private evaluations, sensitive abuse details, donor records, student data, school private data, private telemetry, production prompts, infrastructure details, or sealed YOSO-YAi LLC IP.

Use a private maintainer contact path until a formal Foundation security contact is published.

## Public Issue Scope

Public issues may discuss:

- broken links;
- safety-note wording;
- limitation taxonomy improvements;
- review-gate clarity;
- graph rendering issues;
- public reliance boundary improvements.

Public issues must not include:

- credentials, tokens, or API keys;
- private evaluation detail;
- private red-team notes;
- operational misuse instructions;
- donor, volunteer, student, school, customer, or private partner data;
- private model, dataset, or Space development material;
- security-sensitive NEURONA operational information.

## Supported Scope

This repository is documentation-only. It does not collect data, run production systems, host models, publish datasets, or operate public demos.
